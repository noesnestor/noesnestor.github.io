¿Qué es Machine Learning?

Analizaremos el significado de Machine Learning en base a distintas fuentes:

Cambridge University

The process of computers changing the way they carry out tasks by learning from new data, without a human being needing to give instructions in the form of a program

IBM

Machine learning is a branch of artificial intelligence (AI) and computer science which focuses on the use of data and algorithms to imitate the way that humans learn, gradually improving its accuracy

Google

El aprendizaje automático es un subconjunto de inteligencia artificial que permite que un sistema aprenda y mejore de forma autónoma mediante redes neuronales y aprendizaje profundo, sin tener que ser programado explícitamente, a través de la ingesta de grandes cantidades de datos.

¿Qué tiene en común y en qué se diferencia de “Inteligencia Artificial”?

Machine Learning es un subconjunto dentro de la Inteligencia Artificial, ya que no todas las actividades de Inteligencia Artificial son Machine Learning, pero Machine Learning es en su totalidad Inteligencia Artificial. ¿Qué tiene en común y en qué se diferencia de “Análisis Estadístico”?

La diferencia es en la utilidad que tiene cada una. El análisis estadístico construye modelos que permiten explicar relaciones entre variables, mientras que Machine Learning puede construir modelos para predecir el futuro sin programación explícita. Sacado de: https://www.coursera.org/mx/articles/statistical-modeling

¿Cómo se diferencia con Data Mining?

La diferencia existe en la utilidad que tiene cada una. Data Mining está diseñado para extraer información importante de una cantidad vasta de información, mientras que Machine Learning entrena un sistema usando datos extraidos para realizar tareas complejas. Sacado de: https://www.simplilearn.com/data-mining-vs-machine-learning-article

¿Qué es CRISP-DM?

CRISP-DM

CRISP-DM, o Cross Industry Standard Process for Data Mining, es un modelo de proceso de Data Mining. Describe las fases del ciclo de vida de Data Mining, conjunto con las tareas necesarias para cada fase y las relaciones entre fases.$^1$ Cabe destacar, que aunque desarrolle las tareas, no desarrolla cómo realizarlas.$^2$

Ciclo de Vida

Contiene seis fases con flechas para representar las dependencias entre fases, permitiendo la capacidad de volver hacia atras entre fases si lo fuese necesario. Las fases son:

  • Comprensión del Negocio/Business Understanding
  • Comprensión de los Datos/Data Understanding
  • Preparación de los Datos/Data Preparation
  • Modelado/Modeling
  • Evaluation/Evaluación
  • Deployment/Implementación

Procesos Similares

SEMMA

Es una metodología que se define como “el proceso de selección, exploración y modelado de grandes volúmenes de datos para descubrir patrones de negocios desconocidos”$^3$ creado por el Instituto SAS. Fue creado para trabajarse con el software de su creador en mente, organizando nodos según las fases que componen la metodología. Tiene la ventaja de que “el usuario podría hacer uso del mismo siguiendo cualquier otra metodología de minería de datos”$^4$. Lo cual permite que uno pueda usar metodologías como CRISP-DM dentro de SEMMA.

Tipos de Algoritmos y las herramientas que los soportan

Rapidminer

Clustering/Análisis de Grupos:

  • k means - si - add_cluster_attribute, add_as_label, k, etc
  • k modes - no
  • gaussian mixture model - si, con extension smile
  • dbscan - si - epsilon, minimal_points
  • Hierarchical - si - hierarchy, use_local_random_seed, local_random_seed

Dimension Reduction/Reducción de Dimensionalidad

  • Latent Dirichlet Analysis - No
  • Singular Value Decomposition - Si - dimensionality_reduction, percentage_threshold, dimensions
  • Principal Component Analysis - Si - dimensionality_reduction, variance_threshold, number_of_components

Classification & Regression/Clasificación & Regresión:

  • Kernel SVM - Si - kernel_type, kernel_gamma, kernel_sigma1, etc
  • Linear SVM - No
  • Random Forest - Si - minimal_size_for_split, minimal_leaf_size ,minimal_gain, etc
  • Neural Network - Si - hidden_layers, training_cycles, learning_rate, etc
  • Gradient Boosting Trees - Si - number_of_trees, reproducible, maximum_number_of_threads, etc
  • Decision Tree - Si - criterion, minimal_size_for_split, minimal_leaf_size, etc
  • Logistic Regression - Si - kernel_type, kernel_gamma, kernel_sigma1, etc
  • Naive Bayes - Si - laplace_correction
