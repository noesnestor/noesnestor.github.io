Preparación de los Datos

Posted on mar. 03 Octubre 2023 in Texto de Estudio

Introducción

La preparación es una de las partes fundamentales del modelo de proceso de data mining CRISP-DM. En ella, debemos de analizar si existen modificaciones que puedan mejorar el rendimiento de un posible modelo futuro, y si es así, debemos de aplicarlos.

Para poner en práctica el teórico haremos uso de la herramienta RapidMiner.

Caso de Ejemplo

North (2012) nos propone el siguiente caso:

Jerry es un manager de marketing de una firma pequeña de diseño de internet y publicidad. Su jefe le pide que desarrolle un data set incluyendo información acerca de los usuarios del Internet. La companía utilizará esta información para determinar qué tipos de usuarios utilizan el internet y como la firma puede mejorar su estrategia de marketing para que mejor se acople a los usuarios.

Para hacer esto, Jerry hizo una encuesta online y publica links a la misma en páginas web populares. Una vez pasadas dos semanas, tiene la cantidad de datos suficiente para proseguir al análisis, pero se encuentra que los datos requieren ser optimizados; desde datos inválidos a datos faltantes, Jerry se da cuenta que hay mucho trabajo por hacer aún.

Nótese que en este caso Jerry ya conoce la razón por la cual se obtendrán los datos, y cual es la utilidad que se les quiere otorgar, lo cual son cosas que debemos de saber en el paso de Business Understanding de CRISP-DM.

Además, se conoce la fórma en la cual se conducirá la extracción de los datos y la forma en la cual se almacenarán, cosas que debemos preguntarnos una vez estemos en el paso de Data Understanding de CRISP-DM.

Por ende, lo único que nos queda es proseguir con el tercer paso, que es la preparación de los datos.

Data Scrubbing

Teniendo en cuenta que en el caso ejemplar se nos mencionó que se encontraron datos faltantes y datos inválidos, es momento de empezar a corregir estos datos de forma tal de que vuelvan a cobrar sentido. A esto se lo conoce como Data Scrubbing, y según North (2012), existen cuatro formas de data scrubbing:

  • Manejo de datos faltantes
  • Reducción de los datos
  • Manejo de datos inconsistentes
  • Reducción de atributos

Manejo de datos faltantes

Los datos faltantes son aquellos que, como dice la palabra, no están en nuestro data set. Estos datos generalmente se representan como nulos, y dependiendo de lo que creamos pertinente, la forma en la que los manejaremos. En el caso del data set de Jerry, en RapidMiner veremos estos datos faltantes con la forma del símbolo de pregunta ?:

Esto sería fácil de justificar si nos ponemos a ver la tabla de Other_Social_Network la cual dependiendo de la persona puede ser que tenga más de una red social en uso. Por ende, podríamos justificar que la falta de información ahí es comprensible en base a lo que se pidió del usuario. Por el otro lado, en secciones donde la respuesta es un sí o no, que haya un valor nulo simplemente no tiene sentido. En Read_News, Online_Shopping y Online_Gaming hay datos nulos, los cuales no tenemos idea de cómo interpretar. Para solucionar esto, reemplazaremos los valores nulos por otros valores, lo cual en RapidMiner se hace con el operador Replace Missing Values.

Replace Missing Values

Replace Missing Values es un operador de RapidMiner que al conectarse con un data set, ejecuta una función que toma los valores faltantes y los cambia en base a una función. Estas funciones pueden ser:

  • Para valores numéricos:
    • Mínimo: Los valores se reemplazan por el valor mínimo del atributo que tiene datos nulos.
    • Máximo: Los valores se reemplazan por el valor máximo del atributo que tiene datos nulos.
    • Media: Los valores se reemplazan por el valor resultante del cálculo de la media del atributo que tiene datos nulos.
    • Zero: Los valores se reemplazan por el número cero.
  • Para cualquier tipo de dato:
    • Value: Los valores se reemplazan por un valor definido por el usuario.

Cabe destacar que el usuario tiene la posibilidad de elegir qué atributos quiere que tengan sus datos nulos reemplazados, otorgándo entonces estas opciones, entre otras:

  • All: Todos los atributos del data set tendrán sus datos nulos reemplazados en base a la función elegida.
  • Single: Se elige un atributo para que se aplique la función de reemplazo de datos nulos.
  • Subset: Se elige un conjunto de atributos para que se aplique la función de reemplazo de datos nulos.

Reducción de los datos

El proceso de reducir tus datos implica sacar del data set aquellas filas que nos sean problemáticos a la hora de alimentárselos a un algoritmo de Machine Learning para que nos genere un modelo. Esto lo podemos hacer en RapidMiner utilizando el Operador Filter Examples.

Filter Examples

Este operador contiene muchas variables posibles, por lo cual dejo la documentación aquí. A efectos de este ejercicio, solo nos interesa que reduzca los datos en los cuales el Atributo Online_Shopping tenga un valor nulo. Para esto, debemos de utilizar la condición attribute_value_filter la cual nos deja colocar una condición para que reduzca datos. La condición en cuestión es: Online_Shopping=.. Esta condición busca en el data set las filas que tengan el valor nulo en Online_Shopping, y si tienen, se quitan del data set. Es importante saber que los nulos a la hora de representarlos en la condicional se representan con el punto ., aunque en la visualización se representen con el signo de pregunta.

En el caso de que no tengamos forma de interpretar de forma segura qué son esos datos nulos, simplemente es mejor removerlos antes de agregarles datos. Especialmente si estos datos no son numéricos, donde una función aritmética como la media, moda y mediana podrían sernos útiles.

Manejo de datos inconsistentes

Los datos inconsistentes son datos que, en base a nuestra comprensión de la estructura de los datos, simplemente no tienen sentido. Ahora bien, esto podría abarcar también los datos faltantes, pero hay una clara diferencia, la cual es que en este caso el dato si existe.

Para ver qué tipos de datos tienen cada uno de los atributos, podemos dirigirnos a la sección de Statistics de RapidMiner, la cual es visible una vez ejecutemos el proceso. Una vez ahí, podemos ver un claro ejemplo de un dato inconsistente:

En este caso, por algúna razón, el atributo Twitter tiene un valor 99, lo cual no tiene sentido, ya que solo deberíamos de poder escribir valores de Si o No (Y/N). Aquí tenemos dos opciones:

  • Eliminar toda esa fila de datos
  • Cambiar ese dato inconsistente por otro que tenga más sentido.

A efectos de este procedimiento, reemplazaremos este valor por otro que tenga más sentido. Para esto, agregaremos otro Operador al flujo del proceso, el Operador Replace.

Replace

Similar a Replace Missing Values, tiene los mismos selectores de cantidad de atributos afectados por el operador. La diferencia radica en la forma en la cual actúa. En vez de utilizar funciónes como el promedio, este operador busca todos los valores que coincidan con el input colocado en replace what y los cambia por el input colocado en replace by.

En nuestro caso, solo queremos reemplazar el “99” existente en el atributo Twitter. Para esto, configuraremos el operador Replace para que solo modifique el atributo Twitter, y en el campo replace by, colocaremos el valor 99. Como estamos cambiando los datos, debemos de ser cuidadosos en base a qué valor le colocamos al dato inconsistente. Podemos guiarnos en las gráficas de arriba, las cuales nos dicen que la gran mayoría no usan Twitter, y por ende puede que sea aplicable a esta persona. Por ende, en el valor de replace by colocaremos N.

Reducción de atributos

A la hora de armar una solución a una problemática con Machine Learning, siempre debemos de preguntarnos si en nuestro data set existen datos que simplemente son irrelevantes a la hora de armar la solución. Esto se puede visualizar con métodos que evalúen la correlación entre otros atributos, lo cual no es objeto de estudio para este articulo.

En RapidMiner podemos remover atributos de la solución sin eliminarlos por completo, ya que pueden sernos útiles en otro momento. Para esto, utilizaremos el Operador Select Attributes:

Select Attributes

Este Operador nos permite elegir la cantidad de atributos que queremos que sigan en el flujo del proceso, excluyendo el resto. El Operador tiene dos tipos:

  • include attributes: En base a un filtro, incluirá a todos los que cumplan con él.
  • exclude attributes: En base a un filtro, excluirá a todos los que cumplan con él.

Como podrán ver, hacemos mucha mención de los filtros. Similar a todos los Operadores que vimos, este tiene un input attribute filter type que permite elegir el filtro con el cual se eligirán los atributos que cumplan con él. Estos son los mismos que tiene el Operador Replace Missing Values, y a efectos de este artículo, eligiremos un subconjunto de atributos con los que nos quedaremos, por lo cual attribute filter type tiene de tipo subset.

Al elegir esta opción, se nos muestra un botón para elegir el subset. Clickeamos en él y veremos esta pantalla:

Lo que hacemos aquí es elegir todos los atributos de la izquierda y pasarlos a la derecha. Como el tipo de nuestro Operador es include attributes, todos los atributos que no se seleccionen quedarán afuera del flujo del proceso.

Dependiendo del problema que queremos resolver el conjunto de atributos que seleccionaremos. Para seleccionarlos, clickeamos en el atributo y clickeamos la flecha que va hacia la derecha. Una vez seleccionamos todos los atributos que queremos le damos a Apply y ejecutamos el proceso. La problemática de Jerry era recabar información y mostrarla a la firma, por lo cual no requiere de cambios. Pero en el caso de que quiera conocer en qué redes sociales se encuentran las personas con más tiempo en el internet, puede seleccionar los atributos Years_on_Internet y los distintos atributos que mencionan redes sociales, para obtener la solución a esta pregunta.
